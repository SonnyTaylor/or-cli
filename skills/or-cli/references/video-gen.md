# Video Generation with `or create video`

Generate videos from text prompts using OpenRouter's async video generation API. The command submits a job, polls until completion, and downloads the result.

## Basic Usage

```bash
# Generate and save (default: output.mp4)
or create video "A cat walking on a sunny beach"

# Specify output path
or create video "A time-lapse of flowers blooming" --save flowers.mp4

# With resolution and duration
or create video "A drone shot of mountains" --save mountains.mp4 --resolution 1080p --duration 5
```

## Options

| Flag | Purpose |
|------|---------|
| `-m, --model <id>` | Video generation model (auto-detected if omitted) |
| `--save <path>` | Output file path (default: `output.mp4`) |
| `--duration <seconds>` | Target duration |
| `--resolution <res>` | 480p, 720p, 1080p, 1K, 2K, 4K |
| `--aspect-ratio <ratio>` | 16:9, 9:16, 1:1, 4:3, 3:4, 3:2, 2:3, 21:9, 9:21 |
| `--size <WxH>` | Exact pixel dimensions (e.g. 1280x720) |
| `--frame-image <path>` | First frame image (image-to-video) |
| `--reference-image <path>` | Reference image for style guidance |
| `--no-audio` | Disable audio generation |
| `--seed <n>` | Seed for deterministic generation |
| `--poll-interval <seconds>` | Polling interval (default: 30) |
| `--list-models` | List available video models |
| `--json` | Output metadata as JSON |
| `--quiet` | Suppress non-error output |

## Finding Models

```bash
or create video --list-models       # Dedicated video models endpoint
or models -t video                  # Alternative
or show google/veo-3.1              # Check capabilities
or benchmarks --type text-to-video --sort score -n 10  # Best quality
```

## Image-to-Video

Use `--frame-image` to start the video from a specific image:

```bash
or create video "The character walks forward" --frame-image first-frame.png --save output.mp4
```

Use `--reference-image` for style guidance (the model uses it as visual reference, not an exact frame):

```bash
or create video "A solar flare beside a planet" --reference-image style-ref.png --save output.mp4
```

## How It Works (Async)

Video generation is asynchronous because it takes 30s–several minutes:

1. **Submit** — CLI sends `POST /api/v1/videos`
2. **Poll** — CLI polls `GET /api/v1/videos/{jobId}` every 30s
3. **Download** — Once status is `completed`, CLI downloads from `unsigned_urls[0]`
4. **Save** — Video saved to `--save` path

You'll see status updates as it polls:

```
✔ Job submitted: abc123
  pending
  in_progress (1)
  in_progress (2)
  completed

✓ Saved to output.mp4 (12.3MB)
  Model: google/veo-3.1 • Job: abc123
  Latency: 145s • Polls: 5
  Cost: $0.50
```

## Pricing

Video models charge **per second of video**, not per token. Check with `or create video --list-models` for `pricing_skus`.

## Agent Patterns

```bash
# Generate with JSON metadata
or create video "A sunset" --save sunset.mp4 --json | jq '.cost'

# Quiet mode (suppress progress)
or create video "A cat" --save cat.mp4 --quiet

# Longer poll interval for long videos
or create video "A long scene" --save scene.mp4 --poll-interval 60
```

## Notes

- **Takes 30s–several minutes.** This is normal for video generation.
- **Auto-detects model** if `--model` is omitted.
- **Audio is generated by default.** Use `--no-audio` to disable.
- **Webhooks** are supported on the API but not yet exposed in the CLI. Use `--json` to get the job ID if you need to poll externally.
- **ZDR (Zero Data Retention)** is not supported for video generation due to the async workflow.
